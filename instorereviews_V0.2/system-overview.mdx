# InStore Reviews System Overview

This document provides a detailed explanation of how the InStore Reviews system works, what each component does, and how they all work together. It's written for both technical and non-technical readers.

## What is InStore Reviews?

InStore Reviews is a QR-based customer feedback system that helps businesses collect and manage customer reviews in real-time. When customers scan a QR code at your business location, they can instantly leave feedback, which you can monitor and respond to immediately.

## System Components

### 1. Frontend (Vue 3)
This is what users see and interact with. It includes:

#### Customer-Facing Components
- **Review Form**: Where customers leave their feedback
- **Rating Interface**: Star rating system
- **Comment Section**: For detailed feedback
- **Social Media Sharing**: For positive reviews

#### Business Dashboard
- **Analytics Dashboard**: View feedback statistics
- **QR Code Management**: Create and manage QR codes
- **Location Management**: Handle multiple business locations
- **User Management**: Manage staff access
- **Notification Settings**: Configure alert preferences

### 2. Backend (Laravel)
This handles all the behind-the-scenes processing:

#### Core Features
- **QR System**: Generates and tracks QR codes
- **Review Processing**: Handles feedback submission
- **Notification System**: Sends alerts for new reviews
- **Analytics Engine**: Processes feedback data
- **Authentication**: Manages user access

#### Integration Points
- **SMS Service (Twilio)**: For instant notifications
- **Email Service (Mailgun)**: For email notifications
- **Social Media APIs**: For review sharing

### 3. Database (MySQL)
Stores all system data in an organized way:

#### Main Data Tables
- **Businesses**: Company information
- **Locations**: Store/branch details
- **QR Codes**: QR code data and tracking
- **Reviews**: Customer feedback
- **Users**: System users
- **Roles**: User permissions

## How Everything Works Together

### 1. QR Code Flow
1. Business creates QR codes in dashboard
2. QR codes are placed in store locations
3. Customers scan codes
4. Review form appears on their device
5. Feedback is submitted and processed

### 2. Review Processing
1. Customer submits review
2. System validates input
3. Review is stored in database
4. Notifications are triggered based on rating
5. Analytics are updated in real-time

### 3. Notification System
- **Negative Reviews**: Instant SMS alerts
- **All Reviews**: Dashboard notifications
- **Summary Reports**: Daily email digests
- **Custom Alerts**: Based on business preferences

### 4. Business Management
1. Login to dashboard
2. View real-time analytics
3. Manage multiple locations
4. Configure notification preferences
5. Generate custom reports

## File Structure Explained

### Frontend Files (`resources/js/`)
- `components/`: Reusable UI elements
  - `ReviewForm.vue`: Customer review interface
  - `QrGenerator.vue`: QR code creation tool
  - `Dashboard.vue`: Main dashboard view
- `views/`: Page components
- `store/`: State management
- `utils/`: Helper functions

### Backend Files (`app/`)
- `Http/Controllers/`: Request handlers
  - `QrMediaController.php`: QR code management
  - `ReviewController.php`: Review processing
  - `NotificationsController.php`: Alert system
- `Models/`: Database models
- `Services/`: Business logic
- `Notifications/`: Alert templates

### Common Issues
